Website Encryption

Hide Firebase configs, API keys, code snippets, or any text behind a protected link. Only your allowed domains can read it.

How to use Website Encryption

Hide any text or code behind a protected URL. When fetched from your site, it returns the content automatically — invisible to anyone browsing your source.

Getting started
1
Create a project ▼

Click + New Project, give it a name, and paste what you want to hide — a Firebase config, API key, JS snippet, HTML, anything.

Content is encrypted with AES-256-GCM. The decryption key never touches the server — it lives only in your URL fragment.

2
Set your allowed domains ▼

Add every domain that should be able to fetch the content (e.g. yoursite.com). Leave empty to allow any origin.

When fetched, the page checks document.referrer against your list. No match = empty response.

Note: Referrer-based locking stops casual theft and direct browser access. A determined dev can spoof referrers server-side. For high-value secrets, add Firebase Security Rules as a second layer.
3
Copy your protected link ▼

Your link looks like:

https://htmltools.me/website-encryption?id=abc123xyz#decryptionkey

The #fragment is the decryption key. Browsers strip fragments before sending HTTP requests — so even if someone accesses your Firebase directly, the data is unreadable without it.

Using the link in your code
4
Fetch and use the content ▼

Replace your hardcoded secret with a fetch() call to your protected link. Pick your use case:

// Instead of hardcoding your Firebase config, fetch it: async function loadFirebase() { const res = await fetch('https://htmltools.me/website-encryption?id=YOUR_ID#YOUR_KEY'); const firebaseConfig = JSON.parse(await res.text()); const { initializeApp } = await import('https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js'); const app = initializeApp(firebaseConfig); // use app normally from here } loadFirebase();

Store your config as a JSON string in the project. At runtime it's fetched, decrypted, and parsed — never visible in your source.

// Fetch an API key at runtime async function callApi(prompt) { const apiKey = (await (await fetch('https://htmltools.me/website-encryption?id=YOUR_ID#YOUR_KEY')).text()).trim(); const res = await fetch('https://api.example.com/v1/chat', { method: 'POST', headers: { 'Authorization': `Bearer ${apiKey}`, 'Content-Type': 'application/json' }, body: JSON.stringify({ prompt }) }); return res.json(); }
// Inject protected HTML into the page at runtime async function loadContent() { const res = await fetch('https://htmltools.me/website-encryption?id=YOUR_ID#YOUR_KEY'); document.getElementById('target').innerHTML = await res.text(); } loadContent();

Great for paywalled content, private templates, or anything you don't want in your HTML source.

// Fetch and execute a hidden JS snippet async function loadScript() { const code = await (await fetch('https://htmltools.me/website-encryption?id=YOUR_ID#YOUR_KEY')).text(); const script = document.createElement('script'); script.textContent = code; document.head.appendChild(script); } loadScript();

Caution: Only execute JS you fully trust.

5
What visitors see when they click the link directly ▼

If someone pastes your link directly into a browser or fetches it from an unlisted domain, they get a blank page — nothing. The encrypted blob in Firebase is useless without the #key fragment, and the fragment is never sent over the network.

How it works: Browsers strip #fragments before making HTTP requests. Firebase stores only AES ciphertext. The page reads the key from window.location.hash client-side, checks your domain allowlist, then either decrypts and returns the plaintext or returns nothing.
6
Updating your content ▼

Open the project, edit the content, click Save & Re-encrypt. A new key is generated — update the link anywhere you've pasted it.

The ?id= stays the same. Only the #key changes.
