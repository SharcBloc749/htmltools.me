Website Encryption

Hide Firebase configs, API keys, code snippets, or any text behind a protected link. Only your allowed domains can read it.

How to use Website Encryption

Hide any text or code behind a protected URL. When pasted into your code, it fetches and returns the content automatically — invisible to anyone browsing your source.

Getting started
1
Create a project ▼

Click + New Project in the sidebar, give it a name, and paste whatever you want to hide — a Firebase config, an API key, a JS snippet, raw HTML, anything.

The content is encrypted with AES-256-GCM before being stored. The decryption key never touches the server — it lives only in your URL.

2
Set your allowed domains ▼

In the Allowed Domains section, add every domain that should be allowed to fetch the content (e.g. yoursite.com). Leave it empty to allow any origin.

When the link is fetched, the page checks document.referrer against your list. If it doesn't match, it returns nothing.

Note: Referrer-based locking stops casual theft and browser-based access, but a determined developer can spoof the referrer in a server-side script. For high-value secrets, use Firebase Security Rules as a second layer.
3
Copy your protected link ▼

The link looks like this:

https://htmltools.me/website-encryption?id=abc123xyz#decryptionkey

The part after # is the decryption key. It never gets sent to the server (browsers strip fragments from HTTP requests), so your Firebase data can't be decrypted even if someone accesses the database directly.

Using the link in your code
4
Fetch and use the content ▼

Instead of pasting your secret directly in your code, paste the protected link and fetch it at runtime. Pick your use case:

// Instead of hardcoding your Firebase config, fetch it: async function loadFirebase() { const res = await fetch('https://htmltools.me/website-encryption?id=YOUR_ID#YOUR_KEY'); const configJson = await res.text(); const firebaseConfig = JSON.parse(configJson); const { initializeApp } = await import('https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js'); const app = initializeApp(firebaseConfig); // use app normally from here } loadFirebase();

Store your config as a JSON string in the project. At runtime it's fetched, decrypted, and parsed — never visible in your source.

// Fetch an API key at runtime async function callMyApi(prompt) { const keyRes = await fetch('https://htmltools.me/website-encryption?id=YOUR_ID#YOUR_KEY'); const apiKey = (await keyRes.text()).trim(); const res = await fetch('https://api.example.com/v1/chat', { method: 'POST', headers: { 'Authorization': `Bearer ${apiKey}`, 'Content-Type': 'application/json' }, body: JSON.stringify({ prompt }) }); return res.json(); }
// Inject hidden HTML into the page at runtime async function loadContent() { const res = await fetch('https://htmltools.me/website-encryption?id=YOUR_ID#YOUR_KEY'); const html = await res.text(); document.getElementById('target').innerHTML = html; } loadContent();

Works great for injecting paywalled content, private templates, or anything you don't want visible in your HTML source.

// Fetch and execute a JS snippet dynamically async function loadScript() { const res = await fetch('https://htmltools.me/website-encryption?id=YOUR_ID#YOUR_KEY'); const code = await res.text(); const script = document.createElement('script'); script.textContent = code; document.head.appendChild(script); } loadScript();

Caution: Only execute JS you fully trust — never from an untrusted source.

5
What happens when someone else clicks the link ▼

If someone pastes your link directly into a browser, or fetches it from an unlisted domain, they see a blank page with no content. The encrypted blob in Firebase is unreadable without the key in the #fragment, and the fragment is never sent over the network.

How it works: The #key fragment is stripped by the browser before the HTTP request is made. Firebase stores only the AES-encrypted ciphertext. The page reads the fragment from window.location.hash client-side, decrypts, checks your domain allowlist, then either returns the plaintext or returns nothing.
6
Updating your content ▼

Open the project, edit the content in the Protected content card, and click Save & Re-encrypt. A new key is generated and the link changes — update the link anywhere you've pasted it.

The project ID (?id=...) stays the same. Only the #key fragment changes.
