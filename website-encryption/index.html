Website Encryption

Encrypt any secret — Firebase configs, API keys, snippets — behind a URL. Fetch and decrypt it in your browser with one copy-pasteable snippet. Lock it to specific domains so no other site can use it.

How to use Website Encryption

Encrypt any secret and fetch it from your site with a single async snippet. The decryption key lives only in the snippet — it never touches a server in plaintext. Lock secrets to specific domains so only your sites can use them.

How it works
?
Architecture overview ▼

Your secret is encrypted with AES-256-GCM in the browser before upload. Only the ciphertext is stored — never the plaintext. The decryption key is embedded in the snippet itself.

The snippet fetches the encrypted blob directly from Firebase's REST API, verifies the calling domain against your allow-list, then decrypts it right there in the browser using the Web Crypto API.

Domain locking: When you add allowed domains to a project, the generated snippet includes a domain check. If the page's hostname isn't in your allow-list, the snippet throws an error before decryption — so the key is useless even if someone copies the snippet.

Keep in mind: Anyone who reads your JS source can find the snippet and the key. Domain-locking raises the bar significantly, but a determined person on an allowed domain can still extract the plaintext at runtime. Use this for configs that are annoying to rotate, not for high-value secrets like payment keys.
Getting started
1
Create a project & paste your secret ▼

Click + New Project, give it a name, and paste whatever you want to protect — a Firebase config object, an API key string, HTML, anything.

It's encrypted with AES-256-GCM in your browser before it leaves your device.

2
Add your allowed domains ▼

In the project view, add the hostnames that should be allowed to fetch this secret — e.g. mysite.com or app.mysite.com. The snippet checks window.location.hostname against this list and refuses to decrypt if it doesn't match.

Leave the list empty to allow any domain (useful during development).

3
Copy and use the snippet ▼

Copy the snippet from your project and drop it into your code. It's a plain async function — no imports, no dependencies. Call it, await it, use the result.

The returned value is your original secret as a plain string — parse it, use it as a key, inject it as HTML, whatever you need.

4
Updating your secret ▼

Open the project, paste new content, click Save & Re-encrypt. A new key is generated and the snippet updates — just copy it again. The project ID stays the same.
